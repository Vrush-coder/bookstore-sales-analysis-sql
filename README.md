# 📚 Bookstore Sales Analysis (SQL Project)

## 📌 Project Overview

This project demonstrates **SQL-based data analysis** on a bookstore database.
The goal of the project is to design a relational database, import data, and perform meaningful analysis using SQL queries.

The database contains information about:

* Books available in the store
* Customers who purchase books
* Orders placed by customers

Using SQL queries, the project analyzes **sales performance, customer behavior, and inventory insights**.

---

## 🗂️ Database Schema

The database consists of three main tables:

### 1️⃣ Books

Stores information about books available in the bookstore.

| Column         | Description                        |
| -------------- | ---------------------------------- |
| Book_ID        | Unique identifier for each book    |
| Title          | Name of the book                   |
| Author         | Author of the book                 |
| Genre          | Category of the book               |
| Published_Year | Year the book was published        |
| Price          | Price of the book                  |
| Stock          | Number of books available in stock |

---

### 2️⃣ Customers

Stores details about customers.

| Column      | Description        |
| ----------- | ------------------ |
| Customer_ID | Unique customer ID |
| Name        | Customer name      |
| Email       | Customer email     |
| Phone       | Contact number     |
| City        | Customer city      |
| Country     | Customer country   |

---

### 3️⃣ Orders

Stores purchase details.

| Column       | Description                   |
| ------------ | ----------------------------- |
| Order_ID     | Unique order ID               |
| Customer_ID  | Customer who placed the order |
| Book_ID      | Book purchased                |
| Order_Date   | Date of purchase              |
| Quantity     | Number of books purchased     |
| Total_Amount | Total price of the order      |

---

## 🔗 Entity Relationship

The database follows this relationship structure:

Customers (1) → (Many) Orders
Books (1) → (Many) Orders

This means:

* A customer can place multiple orders.
* A book can appear in multiple orders.

## 📊 SQL Analysis Performed

### Basic Queries

* Retrieve books by genre
* Find books published after a certain year
* Filter customers by country
* Identify orders placed in a specific month
* Calculate total stock of books
* Find the most expensive book

### Intermediate Queries

* Identify books with low stock
* Calculate total revenue
* Retrieve orders with quantity greater than a threshold
* List all available genres

### Advanced Analysis

* Total books sold per genre
* Average price of books in a genre
* Customers with multiple orders
* Most frequently ordered books
* Top expensive books in a genre
* Revenue generated by each author
* Cities where customers spent more than a certain amount
* Customer who spent the most
* Remaining stock after fulfilling all orders

---

## 📈 Example SQL Query

```sql
SELECT b.Title, SUM(o.Total_Amount) AS Revenue
FROM Orders o
JOIN Books b ON o.Book_ID = b.Book_ID
GROUP BY b.Title;
```

This query calculates the **total revenue generated by each book**.
